MVP – DINHEIRO FÁCIL
Aja como um arquiteto de softwares sênior especialista em sistemas financeiros regulamentados, desenvolva um sistema web com arquitetura limpa, modular e escalável para uma gestão completa de empréstimos consignados da empresa Dinheiro Fácil, incluindo:
•	Gestão de clientes
•	Gestão de empréstimos
•	Gestão de comissões
•	Gestão financeira de pagamentos
•	Controle de usuários e permissões
•	Auditoria de ações
•	Dashboard analítico interativo
1. Stack Principal
- Framework: Next.js (App Router obrigatório)
- Backend: Route Handlers (API Routes do App Router)
- Linguagem: TypeScript obrigatório em 100% do projeto
- Estilização: Tailwind CSS
- Componentes: Shadcn UI
- Banco de Dados: PostgreSQL
- ORM: Prisma
- Containerização: Docker + Docker Compose
- Ambiente isolado por containers

2. Arquitetura
- Estrutura modular por domínio
- Separação clara entre:
  	- Camada de apresentação
  	- Camada de aplicação
 	- Camada de domínio
- Camada de infraestrutura
- Utilizar princípios de Clean Architecture
- Utilizar DTOs para entrada e saída
- Validação com Zod
- Nunca misturar lógica de negócio dentro de componentes React

3. Backend
- APIs RESTful
- Uso correto de métodos HTTP
- Tratamento centralizado de erros
- Middleware para autenticação
- JWT para autenticação
- Hash de senha com bcrypt

4. Frontend
- Mobile First obrigatório
- Componentização reutilizável
- Layout com Sidebar + Header
- Estados gerenciados com Server Components sempre que possível
- Evitar uso desnecessário de Client Components

5. Banco de Dados
- PostgreSQL rodando em container separado
- Migrations com Prisma
- Seeds de exemplo

6. Docker
- Container para:
- App
- Banco
- Docker Compose configurado
- Variáveis de ambiente via .env
- Ambiente preparado para produção

7. Entregáveis
Quero que você gere:
1. Estrutura de pastas ideal
2. Arquivo docker-compose.yml
3. Dockerfile
4. Estrutura inicial do Prisma
5. Exemplo de módulo completo (CRUD)
6. Explicação da arquitetura adotada

COMISSÕES
Fluxo:
1.	Gestor seleciona:
o	Usuário
o	Mês/Ano
2.	Sistema gera tabela automática com os registros do período
Em cada registro:
•	Cliente
•	Data
•	Valor líquido
•	Tipo de comissão:
o	🔘 Porcentagem
o	🔘 Valor fixo
•	Campo valor (% ou R$)
•	Valor da comissão (calculado automaticamente)
Ação:
•	✔ Aprovar comissão
Ao aprovar:
•	Registro sai da tela de comissões
•	Vai automaticamente para Financeiro
Regra MVP
•	Comissão percentual = valor líquido × %
•	Comissão fixa = valor informado
•	Registro só entra no financeiro após aprovação
•	Bloquear edição do registro

DEFINIÇÕES IMPORTANTES (PARA EVITAR AMBIGUIDADE DA LLM)

•	"Valor líquido" = valor recebido pelo cliente
•	"Valor bruto" = valor total do contrato
•	"Usuário" = vendedor
•	"Gestor" = usuário nível 1
•	Comissão só existe após aprovação
•	Financeiro só recebe comissão aprovada
•	Registro não pode ser alterado após aprovação da comissão
